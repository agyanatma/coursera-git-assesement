# Contributing

Thank you for your interest in contributing to this project. Contributions of all
kinds are welcome, including bug reports, feature requests, documentation
improvements, and code changes.

## How to Contribute

There are many ways to contribute to this repository:

- Report bugs by opening an issue with clear steps to reproduce the problem
- Suggest new features or improvements through issues or discussions
- Improve documentation for clarity, accuracy, or completeness
- Submit code changes that fix issues or add useful functionality

## Before You Start

Before making significant changes, please:

- Check existing issues and pull requests to avoid duplicate work
- Open an issue to discuss large changes before investing substantial time
- Read the project's documentation and code of conduct

## Development Workflow

To contribute changes:

1. Fork the repository
2. Create a new branch for your work
3. Make your changes in small, focused commits
4. Test your changes locally
5. Submit a pull request with a clear description of what changed and why

## Pull Request Guidelines

When submitting a pull request, please:

- Keep changes focused on a single topic or fix
- Write clear commit messages
- Include tests when applicable
- Update documentation if behavior or usage changes
- Describe the motivation, approach, and any tradeoffs in the pull request

## Coding Standards

Please follow the existing style and conventions used in the repository. If the
project includes formatting, linting, or testing tools, run them before submitting
your changes.

In general:

- Prefer clear, readable, and maintainable code
- Avoid unrelated refactors in the same pull request
- Add comments only when they improve understanding of non-obvious logic

## Reporting Bugs

When filing a bug report, include as much relevant information as possible, such as:

- A short, descriptive title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Screenshots, logs, or error messages if available
- Environment details such as operating system, language version, or browser

## Suggesting Enhancements

Enhancement suggestions are welcome. Please explain:

- The problem you are trying to solve
- Why the proposed change would be useful
- Any possible alternatives you considered

## Review Process

All contributions are reviewed by project maintainers. Feedback may include requests
for changes, clarification, tests, or documentation updates before a contribution can
be merged.

## Code of Conduct

By participating in this project, you agree to follow the repository's code of
conduct. Please review the `CODE_OF_CONDUCT` file for expectations around respectful
and constructive participation.

## Questions

If you have questions about contributing, open an issue and provide enough context for
maintainers to help.
